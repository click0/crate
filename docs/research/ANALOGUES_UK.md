# Найближчі аналоги Crate на FreeBSD

## Ключові характеристики Crate (для порівняння)

1. **Тимчасові jail** — jail створюється під час запуску та знищується після завершення
2. **Самодостатні архіви** — застосунок + залежності запаковані в `.crate` (XZ)
3. **Ізоляція стільничних/GUI-застосунків** — X11, OpenGL, відеопристрої
4. **Мінімальний контейнер** — аналіз ELF через ldd, вилучення зайвих файлів
5. **Декларативна специфікація** — YAML
6. **Простий CLI** — дві команди: `create` та `run`
7. **Автоматична мережа** — VNET + ipfw NAT без ручного налаштування
8. **Написаний на C++17** (~3 000 рядків)

## Докладні описи (за спаданням подібності)

### 1. AppJail — найближчий аналог (22/40)

| | |
|---|---|
| **URL** | https://github.com/DtxdF/AppJail |
| **Мова** | POSIX sh + C |
| **Версія** | 4.9.0 (лютий 2026) |
| **Стан** | Активно розвивається, найбагатший на можливості менеджер jail |
| **Порт** | `sysutils/appjail` |

**Чому він близький до Crate:**
- Явна **«концепція тимчасовості»** (appjail-ephemeral(7)) — jail як «худоба, а не домашні улюбленці»
- **Makejail** — декларативний формат, аналогічний до Dockerfile: `FROM`, `RUN`, `PKG`, `ENTRYPOINT`, `COPY`, `ENV`, `INCLUDE`
- **Архіви `.appjail`** — стиснуті XZ образи з метаданими AJSPEC
- **TinyJails** — експериментальна можливість для мінімальних jail (аналог оптимізації ELF у Crate)
- **Director** — супутній засіб для YAML-описів багатоконтейнерних середовищ
- **Підтримка X11** — монтування `/tmp/.X11-unix` у jail (задокументовано в довіднику)
- **NAT** — через pf, автоматичне керування віртуальними мережами
- 101+ готових файлів Makejail у репозиторії

**Чим відрізняється:**
- Jail є сталими за замовчуванням (тимчасова поведінка — це філософія, а не типовий режим)
- Відсутнє автоматичне прокидання GPU/OpenGL/відео
- Відсутнє керування Xauthority (потрібен ручний `xhost +`)
- 35+ підкоманд проти 2 у Crate
- Shell, а не C++

### 2. Podman + ocijail (19/40)

| | |
|---|---|
| **URL** | https://github.com/dfr/ocijail |
| **Мова** | Go (ocijail), Go (podman) |
| **Стан** | Експериментальний на FreeBSD, активна розробка за підтримки FreeBSD Foundation |
| **Порт** | `sysutils/podman`, `sysutils/ocijail` |

**Чому він близький до Crate:**
- `podman run --rm` — тимчасовий контейнер, знищується після виходу (як `crate run`)
- OCI-образи — самодостатні архіви (пошарові tarball)
- Docker Hub / реєстри образів
- CNI-мережа

**Чим відрізняється:**
- Екосистема OCI (не рідне пакування FreeBSD)
- Відсутній аналіз ELF (повні образи ОС)
- Відсутня підтримка стільниці/GUI
- Потребує root (rootless на FreeBSD відсутній)
- Великий стек залежностей
- Обмежений вибір образів FreeBSD

### 3. Kleene (19/40)

| | |
|---|---|
| **URL** | https://github.com/kleene-project/kleened |
| **Мова** | Elixir (kleened), Python (klee CLI) |
| **Версія** | v0.1.0-rc.1 (передвипуск) |
| **Стан** | Рання стадія, не готовий до промислового використання |
| **Порт** | `sysutils/kleened`, `sysutils/klee` |

**Чому він близький до Crate:**
- `klee run` — створює тимчасовий контейнер з образу, знищує його при виході (семантика Docker)
- Система збирання на кшталт Dockerfile
- PF-мережа з автоматичним налаштуванням

**Чим відрізняється:**
- Клієнт-серверна архітектура (потрібен демон kleened)
- Образи — це набори даних ZFS, а не переносні архіви
- Відсутня підтримка стільниці/GUI
- Відсутній аналіз ELF
- Потребує ZFS

### 4. pot (17/40)

| | |
|---|---|
| **URL** | https://github.com/bsdpot/pot |
| **Мова** | Shell |
| **Версія** | 0.16.1 (березень 2025) |
| **Стан** | Активний, використовується у промисловому середовищі з Nomad |
| **Порт** | `sysutils/pot` |

**Чому він близький до Crate:**
- Стиснуті XZ архіви експорту (знімки ZFS)
- VNET + pf NAT
- **Інтеграція з Nomad** — `nomad-pot-driver` забезпечує тимчасову поведінку (pull -> create -> run -> destroy)
- **Potluck** — реєстр готових образів (50+)
- Тонкі та товсті jail, flavours (шаблони)

**Чим відрізняється:**
- **Потребує ZFS** (не працює на UFS)
- Відсутній власний тимчасовий режим (тимчасова поведінка досягається через Nomad)
- Відсутня підтримка стільниці/GUI
- Відсутній YAML (прапорці CLI + скрипти flavour)
- Архіви залежать від ZFS (не самодостатній rootfs)
- Засіб для інфраструктури/DevOps

### 5. xc (17/40)

| | |
|---|---|
| **URL** | https://github.com/michael-yuji/xc |
| **Мова** | Rust |
| **Стан** | У розробці, презентований на BSDCan |

Контейнерний рушій, орієнтований на FreeBSD. Підтримує OCI-образи та власний Jailfile. Здатний запускати Linux-контейнери через Linuxulator. Демонна архітектура (xcd). Інтеграція з PF, ZFS, DTrace.

**Чим відрізняється від Crate:** демонна архітектура, не зосереджений на стільниці, відсутній аналіз ELF, відсутній у портах.

### 6. Vessel (17/40)

| | |
|---|---|
| **URL** | https://github.com/ssteidl/vessel |
| **Мова** | C + Tcl |
| **Стан** | Побічний проєкт автора |

Контейнери застосунків на FreeBSD. Унікальна особливість: процес `vessel` працює поряд із кожним jail, очікуючи на події через kqueue. VesselFile (синтаксис Tcl) для декларативних описів. Частково написаний на C — єдиний засіб, окрім Crate, що використовує компільовану мову.

**Чим відрізняється від Crate:** не тимчасовий за замовчуванням, Tcl (не YAML), відсутній GUI, відсутні архіви, відсутній аналіз ELF.

## Підходи до ізоляції стільничних застосунків поза jail

### 7. Capsicumizer (15/40)

| | |
|---|---|
| **URL** | https://github.com/myfreeweb/capsicumizer |
| **Стан** | Покинутий (суспільне надбання) |

**«Super Capsicumizer 9000»** — пісочниця на основі Capsicum. Запускає програми в режимі можливостей через LD_PRELOAD (libpreopen). Профілі в синтаксисі UCL (подібно до AppArmor). **Здатний ізолювати GUI-застосунки** (GTK, X11, Wayland).

Єдиний засіб, окрім Crate, спеціально створений для ізоляції стільничних застосунків на FreeBSD. Однак використовує Capsicum (рівень процесу), а не jail (рівень ОС). Не працює з програмами на Go (LD_PRELOAD).

### 8. Curtain / freebsd-pledge (14/40)

| | |
|---|---|
| **URL** | https://github.com/Math2/freebsd-pledge |
| **Стан** | У розробці, потребує перезбирання ядра |

Модуль ядра `mac_curtain` — перенесення `pledge(3)` та `unveil(3)` з OpenBSD на FreeBSD. Утиліта `curtain(1)` запускає програми в пісочниці з обмеженнями файлової системи. Здатний ізолювати Firefox. Однак потребує власного збирання ядра.

## Шаблони (не засоби)

### 9. exec.prepare / exec.release (вбудований jail.conf)

Механізм, вбудований у FreeBSD (12.2+) через хуки `jail.conf(5)`:
- `exec.prepare` — виконується перед створенням jail (наприклад, клонування ZFS із шаблону)
- `exec.release` — виконується після знищення (вилучення клону)

**Найближчий відповідник моделі створити-запустити-знищити** на рівні ядра. Однак потребує ручного налаштування всього іншого.

### 10. MicroJails (шаблон з FreeBSD Wiki)

| | |
|---|---|
| **URL** | https://wiki.freebsd.org/VladimirKrstulja/Guides/MicroJails |

Shell-скрипт Володимира Крстулі: створює порожній корінь jail, копіює `ld-elf.so.1` та лише потрібні бібліотеки `.so` (знайдені через `ldd`), а також мінімальні файли `/etc`. **Це точно та сама техніка**, яку Crate автоматизує в `create.cpp`. Ручна версія головної технічної можливості Crate.

## Найближчі аналоги за окремими характеристиками

| Характеристика Crate | Найближчий аналог |
|---|---|
| **Тимчасова модель** (створити -> запустити -> знищити) | `exec.prepare`/`exec.release` (вбудований jail.conf), Kleene (`klee run`), Podman (`--rm`) |
| **Самодостатні архіви** | Podman (tarball OCI-образів), pot (ZFS -> XZ), AppJail (`.appjail`) |
| **Ізоляція стільниці/GUI** | Capsicumizer (Capsicum + профілі UCL), CBSD (X11 у jail), Curtain (pledge) |

| **Декларативна специфікація** | AppJail Makejail, Focker Fockerfile, Blackship TOML, BastilleBSD Bastillefile |
| **Простота (2 команди)** | Service Jails (1–2 рядки в rc.conf), Capsicumizer (одна команда) |
| **Автоматична мережа** | AppJail (pf + віртуальні мережі), BastilleBSD (pf + rdr-anchor) |

## Висновок

**Жоден окремий засіб не відтворює унікальне поєднання Crate.** Crate займає власну нішу, поєднуючи:
- Тимчасові jail
- Самодостатні XZ-архіви з оптимізацією ELF
- Першокласну підтримку X11/OpenGL/відео
- Специфікації YAML
- CLI із двома командами
- Автоматичну мережу VNET+NAT

Найближчий аналог — **AppJail** (22/40), який поділяє філософію тимчасовості, має формат архівів, TinyJails та підтримку X11, але залишається повнофункціональним менеджером jail із 35+ командами.

Щоб відтворити функціональність Crate за допомогою інших засобів, знадобилося б поєднання:
**AppJail** (концепція тимчасовості + Makejail) + **MicroJails** (техніка ldd) + ручне налаштування X11 + **exec.prepare/exec.release** (хуки jail.conf)
